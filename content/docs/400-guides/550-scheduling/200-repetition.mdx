繰り返しは、ソフトウェア開発におけるエフェクトを扱う際の一般的な要件です。これにより、特定の繰り返しポリシーに従ってエフェクトを複数回実行することができます。

## repeat

`Effect.repeat`関数は、指定されたスケジュールに従って、または最初の失敗が発生するまで、与えられたエフェクトを繰り返す新しいエフェクトを返します。

<Warning>
  スケジュールされた再実行は初期実行に加えて行われます。そのため、
  `Effect.repeat(action, Schedule.once)`は`action`を最初に1回実行し、
  成功した場合、さらに1回追加で繰り返します。
</Warning>

**成功例**

```ts twoslash
import { Effect, Schedule, Console } from "effect"

const action = Console.log("success")

const policy = Schedule.addDelay(Schedule.recurs(2), () => "100 millis")

const program = Effect.repeat(action, policy)

Effect.runPromise(program).then((n) => console.log(`repetitions: ${n}`))
/*
出力:
success
success
success
repetitions: 2
*/
```

**失敗例**

```ts twoslash
import { Effect, Schedule } from "effect"

let count = 0

// 失敗の可能性があるアクションをシミュレートする非同期エフェクトを定義する
const action = Effect.async<string, string>((resume) => {
  if (count > 1) {
    console.log("failure")
    resume(Effect.fail("Uh oh!"))
  } else {
    count++
    console.log("success")
    resume(Effect.succeed("yay!"))
  }
})

const policy = Schedule.addDelay(Schedule.recurs(2), () => "100 millis")

const program = Effect.repeat(action, policy)

Effect.runPromiseExit(program).then(console.log)
/*
出力:
success
success
failure
{
  _id: 'Exit',
  _tag: 'Failure',
  cause: { _id: 'Cause', _tag: 'Fail', failure: 'Uh oh!' }
}
*/
```

### 最初の繰り返しをスキップする

最初の繰り返しをスキップしたい場合は、`Effect.schedule`を使用できます：

```ts twoslash
import { Effect, Schedule, Console } from "effect"

const action = Console.log("success")

const policy = Schedule.addDelay(Schedule.recurs(2), () => "100 millis")

const program = Effect.schedule(action, policy)

Effect.runPromise(program).then((n) => console.log(`repetitions: ${n}`))
/*
出力:
success
success
repetitions: 2
*/
```

## repeatN

`repeatN` 関数は、指定されたエフェクトを指定された回数だけ、または最初の失敗まで繰り返す新しいエフェクトを返します。繰り返しは初回の実行に加えて行われるため、`Effect.repeatN(action, 1)` は最初に `action` を1回実行し、その後成功すればもう1回繰り返します。

```ts twoslash
import { Effect, Console } from "effect"

const action = Console.log("success")

const program = Effect.repeatN(action, 2)

Effect.runPromise(program)
/*
出力:
success
success
success
*/
```

## repeatOrElse

`repeatOrElse` 関数は、指定されたエフェクトを指定されたスケジュールに従って、または最初の失敗まで繰り返す新しいエフェクトを返します。失敗が発生した場合、失敗の値とスケジュールの出力が指定されたハンドラーに渡されます。スケジュールされた繰り返しは初回の実行に加えて行われるため、`Effect.repeat(action, Schedule.once)` は最初に `action` を1回実行し、その後成功すればもう1回繰り返します。

```ts twoslash
import { Effect, Schedule } from "effect"

let count = 0

// 失敗の可能性があるアクションをシミュレートする非同期エフェクトを定義する
const action = Effect.async<string, string>((resume) => {
  if (count > 1) {
    console.log("failure")
    resume(Effect.fail("Uh oh!"))
  } else {
    count++
    console.log("success")
    resume(Effect.succeed("yay!"))
  }
})

const policy = Schedule.addDelay(
  Schedule.recurs(2), // 最大2回繰り返す
  () => "100 millis" // 繰り返しの間に100ミリ秒の遅延を追加
)

const program = Effect.repeatOrElse(action, policy, () =>
  Effect.sync(() => {
    console.log("orElse")
    return count - 1
  })
)

Effect.runPromise(program).then((n) => console.log(`repetitions: ${n}`))
/*
出力:
success
success
failure
orElse
repetitions: 1
*/
```

## 条件に基づく繰り返し

`while` または `until` オプションを使用して、条件によってエフェクトの繰り返しを制御できます。これにより、実行時の結果に基づいて動的に制御することが可能になります。

```ts twoslash
import { Effect } from "effect"

let count = 0

// 各呼び出しで異なる結果をシミュレートするエフェクトを定義する
const action = Effect.sync(() => {
  console.log(`Action called ${++count} time(s)`)
  return count
})

// 特定の条件が満たされるまでアクションを繰り返す
const program = Effect.repeat(action, { until: (n) => n === 3 })

Effect.runFork(program)
/*
出力:
Action called 1 time(s)
Action called 2 time(s)
Action called 3 time(s)
*/
```

<Info>
  エラーに基づいて条件を設定する場合は、成功した結果ではなく
  [Effect.retry](../error-management/retrying#retrying-based-on-a-condition)
  の使用を検討してください。
</Info>
