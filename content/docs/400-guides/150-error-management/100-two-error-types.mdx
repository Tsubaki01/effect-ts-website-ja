他のプログラムと同様に、Effectプログラムも予期された理由や予期しない理由で失敗することがあります。
非EffectプログラムとEffectプログラムの違いは、プログラムが失敗したときに提供される詳細にあります。
Effectは、プログラムが失敗した原因に関する情報をできるだけ多く保持し、詳細で包括的かつ人間が読みやすい失敗メッセージを生成しようとします。

Effectプログラムでは、プログラムが失敗する可能性がある方法は2つあります：

- **予期されたエラー**: これらは、開発者が通常のプログラム実行の一部として予期し、期待するエラーです。

- **予期しないエラー**: これらは予期せず発生し、意図されたプログラムフローの一部ではないエラーです。

## 予期されたエラー

これらのエラーは、_失敗_、_型付きエラー_、または*回復可能なエラー*とも呼ばれ、開発者が通常のプログラム実行の一部として予期するエラーです。
これらはチェックされた例外と同様の目的を果たし、プログラムのドメインと制御フローを定義する役割を果たします。

予期されたエラーは、"Error"チャネルの`Effect`データ型によって**型レベルで追跡されます**。

次の例では、プログラムが`HttpError`型のエラーで失敗する可能性があることが型から明らかです：

```ts
Effect<string, HttpError>
```

## 予期しないエラー

予期しないエラーは、_欠陥_、_型なしエラー_、または*回復不能なエラー*とも呼ばれ、開発者が通常のプログラム実行中に発生することを予期しないエラーです。
予期されたエラーとは異なり、これらはプログラムのドメインや制御フローの一部とは見なされません。
予期しないエラーは、チェックされていない例外に似ており、プログラムの期待される動作の範囲外にあります。

これらのエラーは予期されていないため、Effectは**型レベルで追跡しません**。
しかし、Effectランタイムはこれらのエラーを追跡し、予期しないエラーから回復するためのいくつかの方法を提供します。
