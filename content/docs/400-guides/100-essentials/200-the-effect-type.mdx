`Effect<Success, Error, Requirements>`タイプは、ワークフローやジョブを**不変**の値として**遅延評価**で記述するものです。

このタイプはプログラムのロジックをカプセル化し、成功した場合は`Success`タイプの値を提供し、失敗した場合は`Error`タイプのエラーを返します。さらに、プログラムの実行には`Context<Requirements>`というコンテキストデータのコレクションが必要です。

概念的には、`Effect<Success, Error, Requirements>`を次の関数タイプの効果的なバージョンと考えることができます：

```ts
type Effect<Success, Error, Requirements> = (
  context: Context<Requirements>
) => Error | Success
```

しかし、エフェクトは実際には関数ではありません。同期的、非同期的、並行的、およびリソースを使用する計算をモデル化することができます。

## 型パラメータ

`Effect`型には、以下の意味を持つ3つの型パラメータがあります：

- **Success**（成功）。エフェクトが実行されたときに成功する可能性のある値の型を表します。
  この型パラメータが`void`の場合、エフェクトは有用な情報を生成しないことを意味し、`never`の場合は、エフェクトが永遠に（または失敗するまで）実行されることを意味します。
- **Error**（エラー）。エフェクトの実行時に発生する可能性のある予想されるエラーを表します。
  この型パラメータが`never`の場合、`never`型の値は存在しないため、エフェクトは失敗することができないことを意味します。
- **Requirements**（要件）。エフェクトの実行に必要な文脈データを表します。
  このデータは`Context`という名前のコレクションに格納されます。
  この型パラメータが`never`の場合、エフェクトには要件がなく、`Context`コレクションが空であることを意味します。

<Info>
  Effectエコシステムでは、`Effect`の型パラメータがそれぞれ`A`、`E`、`R`と略されることがよくあります。
  これは、成功値の型**A**、**E**rror、**R**equirementsの略称です。
</Info>

`Effect`値は不変であり、すべてのEffect関数は新しい`Effect`値を生成します。

`Effect`値は実際には何も行いません。それらは効果的な相互作用をモデル化または記述する値に過ぎません。

`Effect`は、Effect ランタイムシステムによって外部世界との効果的な相互作用に解釈されることができます。理想的には、これはアプリケーションの単一のエントリーポイントで発生し、そこで効果的な相互作用が開始されます。例えば、プログラムの実行開始点などです。
